Here is the simplest and easiest way to deploy a containerized application on a Kubernetes cluster, using a small HTML web app as the example.


---

✅ Step-by-Step: Deploy a Simple Containerized App on Kubernetes

We will do 4 things:

1️⃣ Create a simple HTML app
2️⃣ Write a Dockerfile & build image
3️⃣ Create a Kubernetes Deployment
4️⃣ Expose it using a Kubernetes Service


---

1. Create a simple HTML app

index.html

<!DOCTYPE html>
<html>
<body>
<h1>Hello from Kubernetes!</h1>
</body>
</html>


---

2. Create a Dockerfile

Dockerfile

FROM nginx:alpine
COPY index.html /usr/share/nginx/html/

This uses nginx to serve your HTML file.


---

3. Build & Push Docker Image

Build Image

docker build -t yourname/kube-demo:v1 .

Push Image to Docker Hub

docker push yourname/kube-demo:v1


---

4. Create Kubernetes Deployment

deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: kube-demo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: kube-demo
  template:
    metadata:
      labels:
        app: kube-demo
    spec:
      containers:
      - name: kube-demo
        image: yourname/kube-demo:v1
        ports:
        - containerPort: 80

Apply it:

kubectl apply -f deployment.yaml


---

5. Expose the App Using a Service

service.yaml

apiVersion: v1
kind: Service
metadata:
  name: kube-demo-service
spec:
  type: NodePort
  selector:
    app: kube-demo
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080

Apply:

kubectl apply -f service.yaml


---

6. Access Your Application

If you're using minikube:

minikube service kube-demo-service

OR open in browser:

http://<node-ip>:30080

You will see:

> Hello from Kubernetes!
